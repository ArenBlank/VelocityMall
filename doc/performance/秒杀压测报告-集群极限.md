# 秒杀压测报告 — 3 节点集群极限

> 2026-05-08 | k6 v1.7.1 | Windows 11, 14核 20GB, JDK 17

## 集群架构

```
k6 → Gateway×3 (8080/8090/8091) → Nacos lb → Seckill×3 (8083/8093/8094) → MQ → Order
                   ↑ 轮询选择                               ↑ Nacos 自动负载均衡
```

- **Docker Desktop**：MySQL(1000连接)、Redis、Nacos、RocketMQ、Nginx
- **宿主机**：Gateway×3 + Seckill×3 + Product + Order + User（JDK 17 直跑）
- **JVM**：`-Xms384m -Xmx512m -XX:+UseG1GC`（8 进程）
- Sentinel：`SECKILL_QPS_LIMIT=100000` 完全解封

## 结论

| 能力 | 状态 | 证据 |
|------|:---:|------|
| 零超卖 | ✅ | 两轮 1000 库存全部精确售出 |
| Sentinel 解封 | ✅ | 0 次 429 |
| Gateway×3 集群 | ✅ | 3 节点轮询均参与服务 |
| Seckill×3 集群 | ✅ | Nacos lb 自动分发 |
| 吞吐峰值 | 2,142 req/s | 单机 8 JVM 极限 |

**瓶颈**：1500 VU 时 47.5% 连接拒绝，根因是 Windows TCP 临时端口耗尽，非应用代码问题。Linux + `tcp_tw_reuse` 可进一步突破。
