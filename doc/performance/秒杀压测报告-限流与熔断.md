# 秒杀压测报告 — 限流、防超卖、熔断降级

> 2026-05-06 | k6 v1.7.1 | Windows 11, JDK 17, Spring Boot 3.2.4

## 测试架构

```
k6 → Gateway (Sentinel 限流) → Seckill (Redis Lua) → MQ → Order → MySQL
```

## 第一轮：单用户 1000 VU（限流验证）

| 指标 | 值 |
|------|-----|
| 峰值 VU | 985 |
| 总请求 | 40,422 |
| QPS | 1,992 req/s |
| P95 | 349ms |
| 限流(429) | 40,321 (99.75%) |
| 抢购成功 | 1 |

**结论**：Sentinel 5 QPS/SKU 限流生效，99.75% 被 429 拦截。

## 第二轮：多用户 200 VU（防超卖）

| 指标 | 值 |
|------|-----|
| 峰值 VU | 200 |
| 总请求 | 39,153 |
| QPS | 1,561 req/s |
| P95 | 3.96ms |
| 限流(429) | 39,019 (99.65%) |
| **抢购成功** | **50（库存全部售出）** |
| 售罄 | 66 |
| 重复 | 18 |

Redis 最终库存：**0**，bought Set 恰好 **50** 个用户。**零超卖。**

## 第三轮：混沌工程 500 VU（熔断降级）

Product 服务注入 50% `Thread.sleep(5000)`，Sentinel RT 熔断规则（>200ms 视为慢调用，比例 >50% 则熔断 10s）。

| 指标 | 值 |
|------|-----|
| 峰值 VU | 500 |
| 总请求 | 16,083 |
| QPS | 1,146 req/s |
| P95 | 671ms |
| HTTP 失败率 | **0.01%** |
| **熔断降级** | **15,718 (97.7%)** |
| 正常返回 | 362 (2.3%) |

**结论**：Sentinel 在 1 秒内检测到异常并打开熔断器，Tomcat 线程池未被耗尽。无 Sentinel 时推演：线程池在 2 秒内耗尽 → 503 雪崩。

## 三轮汇总

| 能力 | 轮次 | 状态 | 关键证据 |
|------|:---:|:---:|------|
| Sentinel QPS 限流 | 1, 2 | ✅ | 99%+ 请求 429 拦截 |
| Redis Lua 防超卖 | 2 | ✅ | 50 库存精确抢光，0 超卖 |
| Redis Lua 防重 | 1, 2 | ✅ | SISMEMBER 拦截重复用户 |
| Sentinel RT 熔断 | 3 | ✅ | 1s 内熔断，97.7% fallback，失败率 0.01% |
